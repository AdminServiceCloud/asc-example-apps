# 🤖 AGENTS.md — Регламент репозитория asc-example-apps

Правила для агентов и разработчиков. Общий регламент проекта — [../asc-platform/AGENTS.md](../asc-platform/AGENTS.md), здесь — специфика примеров.

## 🌍 Общие правила

1. Документация — **на русском языке**, с emoji. Комментарии внутри примерных `asc.yaml` — можно на английском (их скопируют в международные пакеты).
2. Задачи — в [ROADMAP](../asc-platform/ROADMAP.md) (пример: `REG-002`), статусы `[PLANING 📝]` / `[IN_PROGRESS 🔧]` / `[DONE ✅]` / `[BLOCKED ⛔]`.
3. Каталог `old/` — только справка. Новые примеры — в корне.

## 📦 Правила для примеров

- Формат манифестов — по доку [📦 package-manager](../asc-daemon/docs/package-manager.md); схемы — в [../registry/schema/](../registry/schema/).
- **Правило корня**: в корне репозитория-пакета — ровно один `asc.yaml` (одно приложение) **или** один `asc.stack.yaml` (стек, объединяющий несколько `asc.yaml` в подкаталогах). Вложенные `asc.yaml` без корневого манифеста не индексируются.
- Каждый пример — в своём каталоге, с минимальным README (что это и как поставить).
- Примеры должны реально устанавливаться: перед коммитом проверка `asc source add file://... && asc install <name>` (когда демон реализован).
- Покрывай типы: `docker`, `native`, `utility` и стеки — примеры служат живой документацией формата.
